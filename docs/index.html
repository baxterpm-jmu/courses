Why Loops Matter

Programming is often about doing the same kind of operation many times. In data work, you might need to:

  • Clean every row in a dataset.
  • Check every file in a folder.
  • Count how many observations meet a condition.
  • Convert a list of raw values into a new list of cleaned values.
  • Search through text for words, names, locations, or patterns.
  • Build a summary from many individual records.

A loop lets you express the idea: for each thing in this collection, do something.

Without loops, you would have to repeat yourself manually. That is slow, error-prone, and impossible once your data gets large.

What Is a for Loop?

A for loop repeats a block of code once for each item in a collection.

The general structure is:

for item in collection:
    # do something with item

There are three important parts:

Part Meaning
item A temporary variable name for the current value
collection The thing being looped over
indented block The code that runs once per item
TipPlain-language translation

Read this code:

for fruit in fruits:
    print(fruit)

as:

For each fruit in the collection called fruits, print that fruit.

Naming the Loop Variable


The loop variable can be named almost anything, but it should be meaningful.


These two examples do the same thing:


for fruit in fruits:
    print(fruit)


for x in fruits:
    print(x)


The first version is better because fruit tells the reader what kind of thing is being processed.












ImportantBest practice







Use a singular name for the loop variable and a plural name for the collection.


Examples:


    

  • for student in students:
    • 

  • for score in scores:
    • 

  • for city in cities:
    • 

  • for file in files:
    • 













The Loop Body Must Be Indented


The indented lines belong to the loop. The unindented lines run after the loop is finished.










Notice that the final print() statement only runs once because it is not indented under the loop.








Common Mistake: Incorrect Indentation


Python uses indentation to understand structure. This is different from many other programming languages.


This code is wrong because the loop body is not indented:


for fruit in fruits:
print(fruit)


Python expects an indented block after the colon.












WarningDebugging clue







If you see an error like IndentationError, check whether the lines after your for statement are indented consistently.












Iterating Over Tuples


A tuple is similar to a list, but it cannot be changed after it is created.


You can loop over a tuple the same way you loop over a list.










For basic looping, lists and tuples behave similarly. The major difference is that lists are mutable, while tuples are immutable.








Iterating Over Strings


A string is also iterable. When you loop over a string, Python gives you one character at a time.










This is useful when you need to inspect text character by character.

The range() Function


The range() function creates a sequence of numbers.


It is commonly used when you want a loop to run a specific number of times.


Common patterns:


range(m)          # 0 to m - 1
range(n, m)       # n to m - 1
range(n, m, step) # n to m - 1 by step












ImportantKey idea







range() stops before the ending number.


So range(5) gives:


0, 1, 2, 3, 4


It does not include 5.












range(m): Start at 0


When range() receives one argument, it starts at 0 and stops before that number.










This loop runs five times, but the values are 0, 1, 2, 3, 4.








range(n, m): Choose a Start and Stop


When range() receives two arguments, the first is the starting value and the second is the stopping value.










This produces 1, 2, 3, 4, 5 because the loop stops before 6.








range(n, m, step): Choose the Step Size


The third argument controls how much the value changes each time.










This loop counts by 2.








Counting Backward with range()


The step can also be negative.










Because the stop value is not included, range(5, 0, -1) stops before 0.








When Should You Use range()?


Use range() when you need controlled repetition or numeric positions.


Good uses:


    

  • Repeat something 10 times.
    • 

  • Count from one number to another.
    • 

  • Access list items by position.
    • 

  • Create simulated data.
    • 



Less good use:


    

  • Looping over items when you do not need their positions.
    • 



Compare these two approaches.
















enumerate(): A Better Way to Track Indexes


Sometimes you need both:


    

  • The position of the item.
    • 

  • The item itself.
    • 



For that, use enumerate().










enumerate() is usually cleaner than this:


for i in range(len(data)):
    print(i, data[i])








Starting enumerate() at 1


By default, enumerate() starts counting at 0. You can change that with start=1.










This is useful when creating numbered instructions or reports for humans.








Nested For Loops


A nested loop is a loop inside another loop.










The outer loop controls i. The inner loop controls j.


For every single value of i, the inner loop runs completely.








Understanding Nested Loop Output


Think of nested loops as a table.


If the outer loop has 3 values and the inner loop has 2 values, then the total number of combinations is:


3 * 2 = 6

Off-by-One Errors


Off-by-one errors happen when a loop runs one too many or one too few times.


They are especially common with range().










Remember:


    

  • range(5) gives 0, 1, 2, 3, 4.
    • 

  • It does not include 5.
    • 

  • range(1, 6) gives 1, 2, 3, 4, 5.
    • 









Debugging Strategy: Print the Loop Variable


When a loop is confusing, print the loop variable at each step.










This helps you see what Python is doing during each iteration.








Debugging Strategy: Print Intermediate Results


If a calculation is wrong, print the result as it changes.










This is one of the simplest and most useful debugging strategies.

When writing a loop, ask yourself:


    

  1. What collection am I looping over?
    2. 

  2. What should the loop variable be called?
    3. 

  3. What should happen during each iteration?
    4. 

  4. Do I need to keep track of a total, count, or new list?
    5. 

  5. Do I need an if statement inside the loop?
    6. 

  6. Should I use enumerate() because I need positions?
    7. 

  7. Should I use zip() because I need to combine related lists?

  8. Could a list comprehension make the code cleaner?
